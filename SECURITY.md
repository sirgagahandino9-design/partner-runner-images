## Reporting Security Vulnerabilities

### For Security Issues with Arm-based Runner Images

If you discover a security vulnerability in any Arm-based GitHub Actions runner image, **please report it to the primary repository:**

**👉 [Report to actions/runner-images](https://github.com/actions/runner-images/security/advisories)**

### For GitHub Infrastructure Security Issues

If the issue is related to the GitHub platform itself:
- **Use GitHub's Security Advisory Page:** https://github.com/actions/runner-images/security
- Follow their responsible disclosure guidelines

### For General GitHub Security Questions

- **GitHub Security Advisory:** https://github.com/advisories
- **GitHub Support:** https://support.github.com

## Security Expectations

### What GitHub Maintains

- Security patches for the runner images themselves
- Updates to pre-installed software and tools
- OS-level security updates
- Compliance with GitHub's security standards

### What You Should Do

If you're using Arm-based runner images (`ubuntu-24.04-arm`, `ubuntu-22.04-arm`, or `windows-11-arm`):

1. Keep your GitHub Actions updated
2. Monitor [actions/runner-images](https://github.com/actions/runner-images) for security updates
3. Test image updates in non-critical workflows first
4. Report any security concerns to the primary repository
